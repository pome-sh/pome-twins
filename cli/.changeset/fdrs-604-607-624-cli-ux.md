**CLI front-door fixes: scenario catalog, exit-codes table, and quickstart.**

- `pome scenarios` now lists all bundled scenarios across the **github, stripe,
  and slack** twins (previously only 5 github scenarios were registered; the
  stripe and slack twins were invisible). A test reconciles the catalog against
  the bundled `scenarios/` directory so a new file can't drift out of the
  catalog. `pome init` continues to scaffold the GitHub starter set; Stripe and
  Slack scenarios are copied explicitly with `pome scenarios <twin> --copy`.
  (FDRS-624)
- The README exit-codes reference now documents the full `0–5` contract and
  matches `exitCodeFor` (`cli/src/hosted/errors.ts`) — codes `4` (quota) and `5`
  (usage) were undocumented and `2` was mislabeled. (FDRS-607)
- The bundled triage-agent quickstart now resolves the per-twin
  `.pome-data/github/secret` path generated by `docker compose up`, with a
  smoke workflow and unit test guarding that clean-clone path. (FDRS-604,
  FDRS-647)
- Stale `pome-sh/pome` repository URLs in CLI output and docs (packaging-bug
  report link, scenario report links, install instructions) now point at
  `pome-sh/pome-twins`.
